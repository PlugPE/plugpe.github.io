  • Introduction
  • IMPORTANT: Please make sure to turn on 'Local Multiplayer' in Minecraft PE settings.
    IMPORTANT: Plug needs Minecraft PE 0.12.1 or above.
    Plug for Minecraft PE plugs into Minecraft PE to give you ‘Bukkit’ style commands in your game. Plug works in both single player AND multiplayer games, so both you as the owner of the world, and players who connect are able to use the commands available.

    In this version we’ve added lots of commands you normally only see in the PC version of the game. For a full list check the help section.

    Not only that now you can host your very own world online, using all the same Plug for Minecraft PE commands you are use to. Now you can play with people all over the world and even Voice Chat with them live while you play!

    Please let us know what you think via Twitter and Facebook. Also please leave us a review on the App Store if you like Plug!
  • Getting Started
  • Plug is very easy to use, all you have to do is open Minecraft PE and start a world. Once in the game use commands by typing them directly into the chat.

    Want to play online? - Simple, sign in above, and go to the hosting tab and either edit one of the default configs or create a new one. Choose how and where you want to host (for example online, bluetooth, local wifi), Private/Public and setup your game title and description.

    Not only that but you can also setup permission groups to control who can use certain commands, anti-griefing to prevent players from placing blocks, destroying blocks or using certain items.
    (Just remember you need to select the group you want to use before switching to Minecraft PE. The current group is highlighted in green).

    Want to use Voice Chat? - Simple, just activate Voice by typing '.v on'. (You can find more information about voice commands in the help section under the commands tab).

    Back to command - commands work by typing into the in game chat.

    • For example to show help type:
      .h

    All commands will also show you how to use them if you add a ? to the end.

    • For example to see how to use the Coordinates command type:
      .c ?

    Before we go on, a special note about the .g (Give) command. This command accepts Block/Item IDs OR Names for the type of item you want. Please check the help section (Items tab) for a full list.

    One more special note. All players are given an ID number to make it easier to reference them.

    • To list players type:
      .l

    You can use the ID number in place of the users name in ANY command.

    • For example to give steve (who has ID 1) 3 ironswords either of these will work:
      .g 1 ironsword 3
      .g steve ironsword 3

    Of course you probably don’t want all players to be able to use all commands. For example Kill would allow players to cause chaos!

    To solve this, you can create permission groups and set permissions for each type of player. Just go to the hosting tab and create or edit a config, and you can add a new permission group, or edit the Default groups there. Remember the Default group can’t be deleted, and is the permissions set for ANY new player who joins your world.

    You should leave the permissions for the Default group very minimal, so you have time to figure out who has actually joined, before you give them destructive permissions like Kill.

    Example 1

    • Set Default permissions to only allow the help command. This protects you if a griefer were to join.
    • Create a new group called Normal which has a few more permissions like Give. You can use this group for normal players.
    • Finally create a group called Admin which has all permissions set.

    Now, if Steve were to join, he will only be able to use Help, or message you to ask for more permissions. If Steve doesn’t seem like a griefer you could give them a few more permissions.

    • For example to set Normal permissions:
      .sp [Player] normal

    If Steve turns out to be a good friend of yours you may want to make them an admin and give them permissions to even more commands:

    • For example to set Admin permissions:
      .sp [Player] admin

    Example 2

    • Set Default permissions to only allow the help command. This protects you if a griefer were to join.
    • Create a new group called Creative which has permissions you want to use on a creative world.
    • Finally create a group called Parkour which has permissions you want to use on a parkour style world.

    Then depending on the type of world you are using set each players permission as they join.
    Note: You can create as many groups as you like, and only the owner can set them. So feel free to create a groups for every world you have, and set them as and when you need from inside Minecraft PE. You can delete groups by clicking on them to edit them first.

    • Using Plug
    • Using Plug for Minecraft PE is very easy:

      • Open Plug for Minecraft PE.
      • Open Minecraft PE, press PLAY and open a world.
      • IMPORTANT: Turn on 'Local Multiplayer' in Minecraft PE settings.
      • Open Minecraft PE chat and type .help or .h to see a list of commands.
      • That’s it!

      Remember you can choose how your Plug world is hosted from the Host Server Tab. Create or edit a new config and choose if you want the server hosted online, which permissions you want to use and any anti-griefing measures you want to activate.

      All commands are controlled from inside Minecraft PE chat, just like Plugins on the PC version of Minecraft.

      Plug is a world first and no other App exists like it on the App Store. We are constantly developing it and hope to eventually implement every PC plugin possible!

      If you find a problem with Plug or would just like to give us some feedback, feel free to send us a comment using the email button in the menu above.

      Thank’s for using Plug, we hope you enjoy it!
    • Introduction
    • Don’t have a WiFi connection? That’s ok - Plug works over Bluetooth!

      Now you can play with friends where a WiFi connection is not available like on a road trip. All the same features are available as with WiFi, and connecting up is just as easy...

      (Note: You can actually use WiFi and BlueTooth at the same time if you wish. That means some players can connect over WiFi and others over BlueTooth).
    • Getting Started
    • Connecting with BlueTooth is exactly the same as connecting over WiFi, except you must remember to turn on BlueTooth on your device!

      To host a world:
      • Turn BlueTooth ON on your device - slide up from the bottom of the screen and make sure the BlueTooth icon is selected.
      • Open Plug for Minecraft PE.
      • Open Minecraft PE, press PLAY and open a world. Other players will now be able to connect to your world over BlueTooth

      To join a world that’s being hosted:
      • Turn BlueTooth ON on your device - slide up from the bottom of the screen and make sure the BlueTooth icon is selected.
      • Open Plug for Minecraft PE.
      • Open Minecraft PE and press PLAY. Plug will start searching for nearby devices hosting worlds, this usually only takes 10 seconds or so, but could take up to 60 seconds.
      • Available worlds will appear in the list as blue. Select one, and you will join the game.
      Note: Since Plug does all the work in making BlueTooth connections work, EVERY player who wants to connect over BlueTooth needs to run Plug. That includes the person hosting, AND everyone who wants to join.
    • Commands Reference
    • A lot of commands are available in Plug, and we have done our best to make them as simple to use as possible. If you play the PC version of Minecraft you are probably use to a lot of the commands here.

      Remember you can get a quick usage guide inside Minecraft PE by tying a command followed by a question mark, for example: .c ?
    • Set Permission (.sp)
    • This command sets the permissions for a player by assigning them a Permission Group. Check the Home Tab for more information on how to use this command.

      NOTE: Only the world owner can use this command.

      • Usage: .sp | .sp [Player] {PermGroup}

      To list available permission groups:
      • .sp

      To view which permission group player 'Steve' is in:
      • .sp steve

      To set permission group 'Default' for player 'Steve'
      • .sp steve default
    • Kill (.k)
    • This command kills a player immediately. The player is still able to respawn as normal. Use this command to warn users.

      • Usage: .k [Player]

      To kill player 'Steve':
      • .k steve
    • List (.l)
    • This command lists all the players in the world, which permission group they are in and if they have any special modes set. It also shows the ID numbers for each player. IDs can be used in place of the player name in ANY command.

      Special Modes
      LO = Locked
      IN = Invisible

      • Usage: .l

      To list all players:
      • .l
    • Info (.i)
    • This command shows the world type, seed, number of players, and if any global modes are on

      • Usage: .i

      To show world info:
      • .i
    • Coordinates (.c)
    • This command shows your coordinates, or those of another player, or of all players.

      • Usage: .c | .c [Player] | .c all

      To show your coordinates:
      • .c
      To show coordinates of player Steve:
      • .c steve
      To show coordinates of all players:
      • .c all
    • Experience (.xp)
    • This command gives you or another player Bottles o Enchantment which can be thrown into the ground to gain experience instantly.

      • Usage: .xp || .xp [Player]

      To give yourself experience:
      • .xp
      To give player Steve experience:
      • .xp steve
      NOTE: This is a shortcut to give/be given bottle o enchantment to quickly gain experience. You/The player needs to select the bottles from the hotbar and throw them into the ground to actually gain experience.
    • Effect (.ef)
    • This command applies a potion effect to you or another player.

      • Usage: .ef [Effect] || .ef [Player] [Effect]

      To give yourself Night Vision:
      • .ef NightVision
      To give player Steve use Night Vision:
      • .ef steve Night Vision
      To list all possible effects:
      • .ef
    • Give (.g)
    • This command gives you or another player an item or block. Please see the 'Items' tab above for valid values. (You can also give yourself enchanted items!)

      • Usage: .g [item:type]{,enchantment:level,...} {qty} || .g [Player] [item:type]{,enchantment:level...} [qty]

      To give yourself 1 iron sword use either:
      • .g ironsword
      • .g 267
      To give yourself 3 iron swords use either:
      • .g ironsword 3
      • .g 267 3
      To give 3 iron swords to player Steve use either:
      • .g steve ironsword 3
      • .g steve 267 3
      Some items have different types:
      • .g spawnegg:zombie
      If you don't know the types, Plug will tell you what the options are. For example this command will tell you the various types of spawn egg available:
      • .g spawnegg
      You can also give yourself enchanted items. To give yourself 1 iron sword with enchantment Power level 2:
      • .g ironsword,power:2,punch:4
      To see all possible enchantments:
      • .g ironsword,help
    • Lock (.lo)
    • This command locks a player, or all players preventing them from moving. This is useful if you want to change something in a world without interference.

      • Usage: .lo | .lo [All |Player] [ON|OFF]

      To show your lock status:
      • .lo
      To lock player steve:
      • .lo steve on
      To lock all players
      • .lo all on
    • Invisible (.in)
    • This command turns your or another player invisible.

      • Usage: .in | .in me [ON|OFF] | .in [Player] [ON|OFF]

      To show who is invisible:
      • .in
      To become invisible:
      • .in me on
      To become visible:
      • .in me off
      To make steve invisible:
      • .in steve on
      To make steve visible:
      • .in me off
    • Kick (.kick)
    • This command allows you to kick a player. You may also specify a reason as to why they were kicked.

      • Usage: .kick [Player] {reason}

      To kick steve:
      • .kick steve

      To kick steve with the reason 'you were griefing':
      • .kick steve you were griefing
      NOTE: A kick is just a warning, the player can reconnect at any time. Use .ban for something more permanent.
    • Ban (.ban)
    • This command allows you permanently ban a user. The user will not be able to join ANY of your worlds again while Plug is running. Use .unban to remove the ban.

      • Usage: .ban | .ban [Player] {reason}

      To see the ban list:
      • .ban

      To ban steve:
      • .ban steve

      To ban steve with the reason 'you were griefing':
      • .ban steve you were griefing
      NOTE: A ban uses both their name AND their device id. This means BOTH users with the same name and ANY users from that device will be banned.
    • Unban (.unban)
    • This command allows you remove a ban your previously set.

      • Usage: .unban [Player]

      To unban steve:
      • .ban steve
      NOTE: Use .unban to see the ban list.
    • Message (.msg)
    • This command allows you to send a private message to another player. Only you and the player you send this message to can see this message.

      • Usage: .msg [Player] [message]

      To send steve the message 'where are you?':
      • .msg steve where are you?
      NOTE: To start a chat conversation see the .chat command.
    • Chat (.chat)
    • This command allows you start a private chat with another player. All chat messages will go directly to this player ONLY.

      • Usage: .chat off | .chat [Player] [message]

      To start a private chat with steve use the following command, then chat as normal.
      • .chat steve

      To stop the private chat:
      • .chat off
    • Player Join Limit (.pl)
    • This command allows you to set a player join limit. Only the number of players you set will be allowed to join.

      • Usage: .pl [off] | [No of Players]

      To check the current limit:
      • .pl

      To set the limit to 3 for example:
      • .pl 3

      To turn the limit off:
      • .pl off
      NOTE: If you set the limit to 1, no other players will be able to join.
    • Friend Request (.fr)
    • This command sends a friend request to another online player. Remember you can manage your friends in the app as well by tapping your username in the title bar above, then tap 'Manage Friends'.

      • Usage: .fr

      To send steve a friend request:
      • .fr steve
    • Friend Accept (.fa)
    • This command accepts a friend request from another online player. Remember you can manage your friends in the app as well by tapping your username in the title bar above, then tap 'Manage Friends'.

      • Usage: .fa

      To accept steve's request:
      • .fa steve
    • Voice Chat On/Off (.v)
    • This command turns on your Voice Chat. If you are the world owner, your microphone and speaker are switched on right away. Otherwise a request is sent to the world owner first.

      • Usage: .v on | off

      To turn on or request Voice Chat:
      • .v on

      To turn off Voice Chat, or to cancel your Voice Chat request:
      • .v off
    • Voice Chat List (.vl)
    • This command lists the current status of Voice Chat for all players. Remember only online players can use Voice Chat. 'NOT SUPPORTED' means the player is connected over Bluetooth or WiFi and not the internet.

      • Usage: .vl

      To list the current Voice Chat status:
      • .vl
    • Voice Chat Accept (.va)
    • This command accepts a Voice Chat request. Only the world owner may use this command.

      • Usage: .va [Player]

      To accept a request from steve:
      • .va steve

      To view requests or the general status of Voice Chat use the .vl command.
      • .vl
    • Voice Chat Decline/Disable (.vd)
    • This command declines a Voice Chat request OR disables Voice Chat for a player already chatting. You may also pass an optional reason when using this command.

      • Usage: .vd [Player] {Reason}

      To decline or disable steve's Voice Chat:
      • .vd steve

      To view requests or the general status of Voice Chat use the .vl command.
      • .vl
    • Voice Chat volumes (.vd)
    • This command alters the Voice Chat volume. Remember this is independant from the volume control on your device. You should set the volume on your device to the level you like for MCPE sounds. Then adjust the Voice volume using the .vv command to get the right balance.

      • Usage: .vv [volume]

      To change the volume to 5 (valid values are 0 to 10):
      • .vv 5

      To check the current volume:
      • .vv
